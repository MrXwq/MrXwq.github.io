学无止境之git初识

发表于 2019-04-30 | 更新于 2019-06-10 | 评论数：
本文字数： 2.2k | 阅读时长 ≈ 2 分钟

创建一个文件：

touch 你要建的文件名字

提交到暂存区：

git add “要提交的文件名”

将暂存区的文件提交到git仓库：

git commit -m ‘添加描述’

查询仓库状态：

git status

添加到远程仓库：

git push

复制仓库地址，然后用git下载：

git clone github下复制的地址

  1. 完整的把远程库下载到本地
  2. 创建origin远程地址别名
  3. 初始化本地库

如果你clone了别人的仓库，这个时候你发现你可以更新他的代码，这个时候你可以git push origin master，这个时候你没有权限，别人会收到你的请求，他要是觉得你的好的话，他就会邀请你进入他的分支，这个时候你再git push origin master就可以了

pull = fetch + merge

原作者：

  1. git fetch抓取新内容进本地
  2. git merge合并

如果操作比较简洁的话，直接pull就行了

如果你下的那个仓库有更新，

那么只要git pull 就能把它同步了

三个区域，一个工作区，一个暂存区，一个git仓库

git commit -m “提交描述”

ls:查看当前目录中的所有文件

pwd:查看当前目录

在文件中设置用户名：git config -D user.name ‘你的github名字’

设置邮箱：git config -D email.name ‘你的邮箱’

新建文件夹：mkdir test

在文件内初始化git

连接github

  1. 在Git-Bash-Here 下输入 ssh-keygen获取密钥的地址
  2. 在GitHub上setting下，点SSH and GPG keys下复制你的密钥地址去 SSH keys上

在项目中 打开Git-Bash-Here 输入git init初始化一个仓库（暂存区，这相当于你的一个工作区域，有什么修改都会记录下来）

如果没有显示.git文件，那么设置打开隐藏文件

在github上创建了文件后，在本地电脑上创建一个文件夹和github连接

步骤应该是这样的
  1. 在github创建仓库
  2. 到本地项目打开git Brush
  3. git init
  4. git add .
  5. git commit -m’描述’
  6. git remote add origin git@github.com:MrXwq/king.git(这个是你仓库的地址)
  7. git push -u origin master
  8. 完成
  1. git remote add origin github复制的地址
  2. git pull origin master （下载github上的文件）
  3. touch 你建的文件名（比如：index.html）
  4. git add ‘你要提交的文件名’（可以多个）如果要将整个目录的文件都推到暂存区，可以git add .
  5. git reset HEAD（如果你这个时候后悔了，那么可以使用这个命令返回到原来的状态，也可以指定回复的文件名）
  6. git commit -m ‘你的描述’ （提交到git仓库里）
  7. git push -u origin master（提交到远程仓库）

查看你的git配置

git config –list

查看历史提交

git log

查看完要退出 q

版本对比

git diff

更改提交的备注

git commit –amend

(这样历史提交就不会多一个了)

恢复工作区删除的文件 从暂存区拷贝到工作目录

git checkout – 你的文件名

删除文件

git rm 文件名

git rm -f 文件名（会把暂存区和工作区的文件都删除）

git rm –cached 文件名 （只删除暂存区的文件 ）

修改文件名

git mv 一开始的文件名 修改后的文件名

分支

创建分支

git branch 分支名字

切换分支

git checkout 分支名

创建并切换到分支

git checkout -b feature2

查看所有分支

git branch -v

查看所有分支修改记录

$ git log --decorate --oneline --graph --all

分支分布

Master: 用来发布的

Hotfix: 用来修补程序的bug

Release：内部发布版 （测试完就发布）

Develop：开发版 （开发完之后编译成可执行程序）

Feature：相关的功能

合并分支

git merge feature

删除分支

git branch -d 分支名

邀请别人进分支

  1. 在我们仓库中，点Settings
  2. 点Collaborators 输入你想邀请的人的账户
  3. 复制连接 发送给那个人
  4. 那个人点击网页，然后点接受

如果想有自己的代码想合并到别人的代码中

先fork，在自己的远程仓库fork进来了一份，然后git clone下来本地，对他进行了一系列操作，操作之后，上传到自己的远程仓库，Pull requests 点击 Create pull request,

这个时候，原作者审核代码，互相交流，最后再合并。
