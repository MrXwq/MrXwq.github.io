Cookie

限制

  1. 每个浏览器都有数量限制
  2. 每个Cookie都有大小限制，一般都在4KB左右，超过大小则无效

构成

  1. 名字：不区分大小写
  2. 值：
  3. 域：domain = “”
  4. 路径：path = “” 指定那个路径应该向服务器发送cookie
  5. 失效时间：expires = “” GMT格式
  6. 安全标志：secure 加了这个，http请求不能发送cookie，https中才可发送
  7. httpOnly：前端无法操作cookie
方法
服务端设置 cookie

不管你是请求一个资源文件（如 html/js/css/图片），还是发送一个ajax请求，服务端都会返回response。而response header中有一项叫set-cookie，是服务端专门用来设置cookie的。如下图所示，服务端返回的response header中有5个set-cookie字段，每个字段对应一个cookie（注意不能将多个cookie放在一个set-cookie字段中），set-cookie字段的值就是普通的字符串，每个cookie还设置了相关属性选项。

客户端发送cookie

客户端可以设置cookie 的下列选项：expires、domain、path、secure（有条件：只有在https协议的网页中，客户端设置secure类型的 cookie 才能成功），但无法设置HttpOnly选项。

demo.appcache

cached:当离线资源存储完成之后触发这个事件
checking:当浏览器对离线存储资源进行更新检查的时候会触发这个事件
downloading:当浏览器开始下载离线资源的时候会触发这个事件
progress:当浏览器在下载每一个资源的时候会触发这个事件，每下载一个资源就会触发一次。
updateready:当浏览器对离线资源更新完成之后会触发这个事件
noupdate:当浏览器检查更新之后发现没有资源更新的时候触发这个事件
