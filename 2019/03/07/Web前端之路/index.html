首先是数组对象

typeof()返回类型

a instanceof Object 返回true 或 false

创建数组

var array = new Array()

var array = new Array(20)

var array = []

判断：

array.isArray(value)

转换方法：

array.valueOf()返回数组本身

array.toString()返回数组中的每一项，每一项中带逗号（jion（）方法可以自定义带什么符号），返回一个字符串

推入栈方法：

push()可以添加任意数量的参数，将他添加到数组的末尾，并返回新数组的长度，改变原来数组的值

pop()可以删除最后一项，并返回删除的元素

shift()删除数组的第一项，并返回删除的元素

unshift()添加任意项在前端，并返回新数组的长度，改变原数组

排列方法：

reverse()返回相反的数组，对原数组产生影响

sort()升序排列数组项，对原数组产生影响，对Unicode编码排序，给它传一个参数，这个参数是函数，函数里面有两个参数,浏览器会根据回调函数的返回值来决定元素的顺序，若返回大于零的值，则元素交换位置，若小于零则位置不变，零也不变。

乱序排序
操作方法

concat()创建当前数组的一个副本，再将其参数加入到该副本，然后返回，传入多少参数都会加入新数组

slice()有两参数，一个是起始位置，一个是结束位置，起始位置算入，而末尾位置不返回，不会影响原数组

splice()

删除的话，只需两个参数，第一个，是位置，从哪开始删，第二个是个数，删除多少个参数

插入的话，任意参数，但前两参数跟前面的一样，后面的参数都将插入到删除之后的数组，若第二个参数为零的话，那么就不删除，直接添加。会影响原数组，原数组为删除后的，返回的为删除的元素。

查找位置方法

indexOf()有两个参数，第一个参数是你要查的数，第二个参数是你要从什么位置开始查找，从头开始查找(可以用来做查找重复)

lastIndexOf()一样，只是从末尾开始向前找

没有找到的话返回-1

迭代方法 IE8及一下都不兼容

这几种方法都是对数组的遍历，里面的参数都是一个函数，函数里都是三个参数，第一个该数组的值，第二个该数组的值的位置，第三个，该数组

some()，遍历所有，若有返回true的，则返回true

every()，遍历数组，遍历到只要有不返回true的则返回false

filter(),遍历所有，返回该数组符合条件的所有值

map(),遍历所有，返回根据公式改变后的数组

forEach(),遍历所有，然后在里面做其他相关的事

上面的值都不会修改原数组的值

归并方法

reduce()里面有两个参数，一个是函数，另一个是初始值(函数里的第一个参数值)只用一次，函数里有四个参数，第一个是前一个值（若没有定义reduce的第二个参数，则第一个参数为数组里的第一个值），第二个是当前值(如果没定义reduce的第二个参数，则为数组里的第二个值)，第三个是值的索引值，第四个是该数组；

document.write会重绘整个页面

document.innerHTML只会重绘所指定的元素内容

回调函数

由我们创建，但不由我们调用

join()

将数组转换为字符串里面的参数是连接符，若不指定连接符则为逗号

[].slice.call()

这个可以让一个伪数组拥有真正的数组方法。

终于知道为什么是要加slice了，原来不加参数是等于复制这个数组了

类（伪）数组使用数组方法

Javascript中存在一种名为伪数组的对象结构。比较特别的是 arguments 对象，还有像调用 getElementsByTagName , document.childNodes 之类的，它们返回NodeList对象都属于伪数组。不能应用 Array下的 push , pop 等方法。
但是我们能通过 Array.prototype.slice.call 转换为真正的数组的带有 length 属性的对象，这样 domNodes 就可以应用 Array 下的所有方法了。
