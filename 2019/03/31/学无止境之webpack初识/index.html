学无止境之webpack初识

发表于 2019-03-31 | 更新于 2019-04-02 | 评论数：
本文字数： 1.7k | 阅读时长 ≈ 2 分钟
webpack安装

全局： npm i webpack -g

项目安装：npm i webpack –save-dev

文件名别webpack

在webpack1文件中创一个src文件夹，放我们的源代码

dist文件夹，我们发布完的都放dist去，到时直接把dist拷给用户就行

main.js 以后导包都在这import $ from ‘jquery’

vscode中，先npm init -y初始化

然后需要什么包可以直接安装，比如jquery ：npm i jquery -S

终端：

返回上一层目录，cd ..

cls清空

自建一个webpack.config.js:

const path = require(‘path’);

module.exports = {

entry: '.\src\index.js',

mode: 'development',

output: {

    filename: 'main.js',

path: path.resolve(__dirname,’dist’)

}

}

在4.0之后的版本需要npm i -D webpack-cli安装一个命令行工具

需要安装一个生产环境 npm i lodash -P

运行npx webpack执行

vscode

alt+b打开浏览器

ctrl+·打开终端

也可以在webpack.config.js上配置

npm run dev执行

这样，只要代码变了，就不用我们每次都npx webpack执行了；

处理费js烈性的文件，需要手动安装合适的第三方loader加载器
处理css

npm i -D style-loader css-loader

style-loader可以将文件中依赖的css文件生产一个style标签库，最终注入到html中

处理less

先cnpm i less-loader -D

再cnpm i less -D

在内存生成html页面的插件

cnpm i html-webpack-plugin -D把页面也放入到内存里

在webpack.config.js中导入这个插件

只要是插件，都要放入到plugins节点中去

处理图片

ES6中的一些高级语法，webpack还处理不了，需要导入第三方loader包，帮助webpack将高级语法转为低级语法

1.在webpack中可以运行如下两套命令

1.1 第一套包：cnpm i babel-core babel-loader babel-plugin-transform-runtime -D

1.2第二套包：cnpm i babel-preset-env babel-preset-stage-0 -D

2.在配置文件中配置

# -webpack
