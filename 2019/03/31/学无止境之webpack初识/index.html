webpack安装

全局： npm i webpack -g

项目安装：npm i webpack –save-dev

文件名别webpack

在webpack1文件中创一个src文件夹，放我们的源代码

dist文件夹，我们发布完的都放dist去，到时直接把dist拷给用户就行

main.js 以后导包都在这import $ from ‘jquery’

vscode中，先npm init -y初始化

然后需要什么包可以直接安装，比如jquery ：npm i jquery -S

终端：

返回上一层目录，cd ..

cls清空

自建一个webpack.config.js:

const path = require(‘path’);

module.exports = {

entry: '.\src\index.js',

mode: 'development',

output: {

    filename: 'main.js',

path: path.resolve(__dirname,’dist’)

}

}

在4.0之后的版本需要npm i -D webpack-cli安装一个命令行工具

需要安装一个生产环境 npm i lodash -P

运行npx webpack执行

vscode

alt+b打开浏览器

ctrl+·打开终端

也可以在webpack.config.js上配置

npm run dev执行

这样，只要代码变了，就不用我们每次都npx webpack执行了；

处理费js烈性的文件，需要手动安装合适的第三方loader加载器
处理css

npm i -D style-loader css-loader

style-loader可以将文件中依赖的css文件生产一个style标签库，最终注入到html中

处理less

先cnpm i less-loader -D

再cnpm i less -D

在内存生成html页面的插件

cnpm i html-webpack-plugin -D把页面也放入到内存里

在webpack.config.js中导入这个插件

只要是插件，都要放入到plugins节点中去

重新梳理：

webpack是基于Node进行构建的，所以，webpack的配置文件中，任何合法的Node代码都是支持的

首要：

把项目各个目录建好:

​

然后初始化：npm init -y

安装webpack ：npm i webpack webpack-cli(4.0之后需要) -D

(npm i -D 是 npm install –save-dev 的简写，是指安装模块并保存到 package.json 的 devDependencies中，主要在开发环境中的依赖包)

在项目下创建一个webpack.config.js配置webpack

启动devServer需要安装一下webpack-dev-server: npm i webpack-dev-server -D

他自动将打包好的文件放到根目录，bundle.js看不见的

安装插件：

这个插件的两个作用：

1. 自动在内存中根据指定页面生成一个内存页面、
2. 自动把打包好的bundle.js追加到页面中去

如果需要处理非js类型的文件，我们需要手动安装一些合适的第三方loader加载器

css：

npm i style-loader css-loader -D

url地址(图片，字体库):

npm i url-loader file-loader -D

babel：

npm install –save-dev babel-loader @babel/core @babel/cli @babel/preset-env(包含了所有es语法)

npm install --save @babel/polyfill     (这个不用-D)

npm @babel/plugin-proposal-class-properties -D

vue：

npm install vue-loader vue-template-compiler –save-dev

npm i vue-router -S

向外暴露成员

ES6中：

​ 导入：import 名字(这个名字可以任意取)，{title as title123(要取其他名字，需要用as)} from ‘路径’

​ 暴露：export default{} ——只允许有一个

​ export let title = ‘nice！’ ———–可以有多个

Node.js中：

​ 导入：var 名称 = require(‘模块标识符’)

​ 暴露：modules.exports 或者 exports
