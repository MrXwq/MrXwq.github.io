建博客之路

最开始我是从B站看视频和别人学的

最开始的问题

从 hexo d开始，从这一步，我出现的东西就和视频中的不一样了，这个时候，因为我是刚接触，所以我也不太懂到底是哪里错了，我就看是不是配置文件config.yml 那里错了，我试了试，改过很多，也找了一下官网，看看配置问题，做了很多尝试，最终还是错了，所以，我做了很愚蠢的事情，就是把所有东西都删除，重新看着视频来做了一遍，前面都一样，很顺利，但到了hexo d 那又是一样，啥都不弹出来，后来我就看其他的视频，看到了一个问题，原来再github那创建文件那有门道，还得在Settings那设置GitHub Pages设置为master branch，命名也有门道，原来后面还得加github.io，最后我又试了试，还是不行，最后，我觉得，我这么错，肯定也有人也是这么错的，我看到，评论里还真有人错了，所以，我直接打问题在百度，果然一堆回答，原来是配置那：后面得加个空格，也就是少了三个空格，我花了好几个小时。。。不过经过这番折腾，我也慢慢熟悉了，也知道了很多东西。

我最想要的

发表文章

就是在上面发表自己写的文章，这个是我最终的目的吧，让我最烦恼的是，它不像微信，QQ空间啥的，不能直接在网页上发表自己的文章，所以我得寻找，该怎么在自己建的博客上发表，最后，经过一番波折，让我终于是成功的发表了这篇文章，具体是这样的

hexo n “my article” #完整命令为hexo new,用于新建一篇名为“my article”的文章

hexo g #完整命令为hexo generate,用于生成静态文件

hexo s #完整命令为hexo server,用于启动服务器，主要用来本地预览

hexo d #完整命令为hexo deploy,用于将本地文件发布到github等git仓库上

编写about页面

这个页面也是让我极其的烦恼，一开始百度了下，具体步骤：

hexo new page “about”

然后就会在source目录下有个about文件出来，里面就会有个md文件，然后我就在里面开始写我的about页面了，写完之后，运行服务器，结果，还是网页完全没有变化，后来经过各种方法改正，最后百度找到，原来是about里生成的文件名不能改，我一开始手贱改了。。。

编写tags页面

这个页面我觉得，我是真难受，我一直以为是在tags文件里面的md直接写标签出来，就在那一直改一直改，怎么都没变化，一直在百度搜了特别多文章，终于看到了原因，原来是在tags文件里要添加type： “tags”，这只是第一步，还得在你写的文章里添加标签属性，比如：我在我这篇文章就添加了-blog，这个时候，我在tags页面就看到了改变。

设置评论系统

我用的是Valine,第一次设置就成功了，显示进入Leancloud官网，注册了一波，然后创建一个应用，名字随便起了个，注册完后，点击右上角，进入之后，点击应用，下面就有刚才你建的应用，点进去，左边最下面有个设置，点击设置，进去后点击应用Key，里面就有 App ID，App Key；这个时候就进去next文件 修改_config.yml里面的valine

終於完成了這簡單的一些東西了

這是我花了三個晚上弄出來的，一些背景啊，域名啊，啥特效的，我就不去弄了，主要能發文章啥的就好了，這是我的第一個目標，完成自己的博客，接下來就是學其他了，好了，第一篇文章就這樣了，結束一波！
