学无止境之移动端适配

发表于 2019-12-24 | 更新于 2019-12-26 | 评论数：

分辨率

像素

可以作为图片和电子屏幕的最小组成单位

屏幕分辨率

一个屏幕具体由多少个像素点组成

图片分辨率

图片含有的像素数，比如一张图的分辨率为800*400，那么它在垂直和水平的像素点数分别为800和400

同以尺寸的图片，分辨率越高，图片越清晰。

PPI && DPI

每英寸包含的像素点

每英寸包括的点数，一般用来描述打印机：打印点的密度

设备独立像素

DIP或DP（Device Independent Pixels）

上面描述的像素都是物理像素，即设备上真实的物理单元。

Retina Display：视网膜屏幕

设备像素比

dpr（device pixel ratio）物理像素和设备独立像素的比值

web中

css

iphone6/7/8 plus的实际物理像素是10801920 开发者工具中显示：设备独立像素（414 736）dpr（3），设备独立像素dpr并不等于 1080 1920 而是等于1242 * 2208

实际上，手机会自动把1240 2208个像素点塞进 1080 1920 个物理像素点来渲染，而1242 * 2208被称为屏幕的设计像素。我们开发过程也是以这个设计像素为准

web端

在写css时，我们用到最多的单位时px，即css像素，当页面缩放比例为100%时，一个css像素等于一个设备独立像素。但是css像素很容易被改变，当用户对浏览器进行了方法，css像素会被放大，这时一个css像素会跨越更多的物理像素

屏幕

我们经常见到用K和P这个单位来形容屏幕

P：屏幕纵向的像素个数，1920 * 1080的屏幕就属于1080P屏幕

K：屏幕横向有几个1024个像素，横向超过4094就属于4K

视口

viewport，在web中，通常不包括浏览器的UI，菜单栏等

布局视口

layout viewport：当我们以百分比来指定一个元素的大小时，它的计算是由这个元素的包含块计算而来的。当这个元素是最顶级的元素时，它就是基于布局视口来计算的。

pc：布局视口不包括borders、margin、滚动条

移动端：布局视口被赋予一个默认值，大部分为980px

视觉视口

visual viewport：用户通过屏幕真是看到的区域。（包括滚动条）

用户将浏览器窗口放大了200%，这时浏览器窗口中的css像素会随视觉视口放大而放大，这时一个css像素会跨越更多的物理像素。

所以布局视口限制你的css布局而视觉视口决定用户具体看到什么。

理想视口

ideal viewport

网站页面在移动端展示的理想大小。在浏览器调试移动端页面上给定的像素大小就是理想视口大小，它的单位正是设备独立像素

页面的缩放系数 = 理想视口宽度 / 视觉视口宽度更为准确

Meta viewport

value 可能值 描述
width 正整数或device-width 以pixels为单位，定义布局视口宽度
height 正整数或device-height 以pixels为单位，定义布局视口高度
initial-scale 0.0-10.0 定义页面初始缩放比率（理想视口宽度/视觉视口宽度）
maximum-scale 0.0-10.0 定义缩放最小值
minimum-scale 0.0-10.0 定义缩放最大值
user-scalable 一个布尔值（yes或者no） 如果no，用户将不能放大或缩小网页。默认yes

移动端适配

device-width就等于理想视口的宽度，所以设置width=device-width就相当于让布局视口等于理想视口。

由于initial-scale = 理想视口宽度 / 视觉视口宽度，所以设置initial-scale = 1；就相当于视觉视口等于理想视口。

这时，一个css像素就等于1个设备独立像素。

缩放

width可以决定布局视口的宽度，设置initial-scale也影响布局视口，布局视口宽度取的是width和视觉视口宽度最大值

获取浏览器大小

window.innerHeight：获取浏览器视觉视口高度（包括垂直滚动条）

document.documentElement.offsetHeight：布局视口宽度

1px问题

为了设配各种屏幕，我们写代码时一般使用设备独立像素来对页面进行布局。

而在设备像素比大于1的屏幕上，我们写的1px实际上是被多个物理像素渲染，这就会出现1px在有些屏幕上看起来很粗的现象。

目前主流的屏幕DPR=2（iphone8）或者3（8p）。拿2倍屏来说，设备的物理像素要实现1像素，而DPR=2，所以css像素只能是0.5。

vh、vw方案适配

视觉视口宽度window.innerWidth 和 视觉视口高度 window.innerHeight 等分为100份

1vw等于视觉视口的1%

vmin：vw和vh中的较小值

缺点：1. px转换成vw不一定能完全整除，因此有一定的像素差。2. 当容器使用vw，margin采用px是，很容易造成整体超过100vw。解决：使用padding带太，结合calc()函数。

图片模糊

原因

位图由一个个像素点构成，每个像素都具有特定的位置和颜色值

理论上，位图的每个像素对应在屏幕上使用一个物理像素来渲染，才能达到最佳的效果

而在 dpr > 1的屏幕上，位图的一个像素可能由多个物理像素来渲染，然而这些物理像素点并不能被准确的分配上对应位图像素的颜色，只能取近似值，所以相同的图片在dpr > 1的屏幕上会模糊

解决

针对不通过DPR的屏幕我们需要展示不同分辨率的图片

或者使用svg图

单位

em

em作为font-size的单位，其代表父元素的font-size，em作为其他属性单位时，代表自身字体大小。

rem

rem作用于非根元素时，相对于根元素字体大小；作用于根元素字体大小时，相对于其出初始字体大小（看浏览器，一般16px）

rem布局的本质是什么？

其实rem布局的本质是等比缩放，一般是基于宽度

弹性布局和响应式布局

rem是弹性布局的一种实现方式，弹性布局可以算作响应式布局的一种，但响应式布局不是弹性布局，弹性布局强调等比缩放，100%还原；响应式布局强调不同屏幕要有不同的显示，比如媒体查询
